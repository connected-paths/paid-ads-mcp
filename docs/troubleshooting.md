# Troubleshooting

## The MCP server shows as unhealthy in a directory

Some directories and marketplaces test remote MCP servers without user credentials. Paid Ads MCP is an authenticated hosted server, so unauthenticated probes may show as unhealthy even when the service is operating normally.

## I cannot connect my AI client

Check that:

- you are using the correct hosted endpoint: `https://paidads-mcp.connectedpaths.com/mcp`
- your browser sign-in is complete
- your billing entitlement is active
- your Google Ads and/or LinkedIn Ads connections are set up in the account console

## I connected, but tools are unavailable

Possible causes:

- no active billing entitlement
- provider connection expired or was not completed
- host client is not using the correct MCP transport
